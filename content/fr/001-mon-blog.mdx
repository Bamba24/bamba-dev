# Bienvenue sur mon blog de test

Ce blog est un exemple simple pour montrer comment structurer un article en Markdown.

## Pourquoi ce blog de test ?

- Pour vérifier le rendu Markdown
- Pour tester les titres, paragraphes et listes
- Pour s’assurer qu’un article s’affiche bien dans un projet Next.js ou MDX

<Note>
Attention soyez vigilant lors de la rédaction de votre article. Assurez-vous que le contenu est clair et bien structuré pour offrir une bonne expérience de lecture à vos visiteurs.
</Note>

## Contenu de l’article

### Introduction

Ce premier paragraphe présente l’objectif du blog : créer un article clair, lisible et facile à modifier.

### Développement

Voici quelques idées à développer :

1. Présentation du projet
2. Installation et configuration
3. Écriture de contenu en Markdown
4. Mise en page et styles

### Exemple de code

```markdown
## Exemple de code Markdown

- Titre
- Liste
- **Texte en gras**
- *Texte en italique*
```
