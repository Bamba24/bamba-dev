# React 19 : ce qu’il faut retenir

React 19 apporte une série d’améliorations conçues pour rendre les applications plus fluides, plus faciles à maintenir, et mieux intégrées aux frameworks modernes.

## 1. Le rendu concurrent plus stable

Le mode concurrent continue d’évoluer. React 19 améliore la capacité du moteur de rendu à interrompre les tâches longues et à donner la priorité aux interactions utilisateur.

- Rendu non bloquant pour les mises à jour importantes
- Meilleure gestion des transitions d’interface
- Réductions de la latence de réponse utilisateur

### Exemple : `startTransition`

```tsx
import { useState, startTransition } from 'react';

function Search() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState<string[]>([]);

  function handleChange(event: React.ChangeEvent<HTMLInputElement>) {
    const value = event.target.value;
    setQuery(value);

    startTransition(() => {
      const filtered = fetchSuggestions(value);
      setResults(filtered);
    });
  }

  return (
    <div>
      <input value={query} onChange={handleChange} placeholder="Recherche..." />
      <ul>{results.map((item) => <li key={item}>{item}</li>)}</ul>
    </div>
  );
}
```

> Ici, `startTransition` indique à React que la mise à jour des résultats peut être différée si le thread est occupé.

## 2. `useEvent` et gestion d’événements améliorée

`useEvent` permet d’attacher des callbacks d’événements sans recréer inutilement la fonction à chaque rendu.

```tsx
import { useEvent } from 'react';

function FavoriteButton({ onToggle }: { onToggle: () => void }) {
  const handleClick = useEvent(() => {
    onToggle();
  });

  return <button onClick={handleClick}>Ajouter aux favoris</button>;
}
```

Ce hook est utile pour les composants qui reçoivent des callbacks en props et veulent minimiser les re-rendus. Il garantit aussi que le callback utilise toujours les dernières valeurs capturées.

## 3. `useDeferredValue` pour lisser les interfaces

Lorsque vous avez des données lourdes à afficher, `useDeferredValue` permet de retarder une partie du rendu tout en gardant le champ de saisie réactif.

```tsx
import { useDeferredValue, useMemo } from 'react';

function FilteredList({ items }: { items: string[] }) {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);

  const filteredItems = useMemo(
    () => items.filter((item) => item.toLowerCase().includes(deferredQuery.toLowerCase())),
    [items, deferredQuery]
  );

  return (
    <div>
      <input value={query} onChange={(e) => setQuery(e.target.value)} placeholder="Filtrer" />
      <ul>{filteredItems.map((item) => <li key={item}>{item}</li>)}</ul>
    </div>
  );
}
```

## 4. `useSyncExternalStore` pour les sources d’état externes

React 19 continue d’encourager les bibliothèques réactives en normalisant l’accès à des stores externes.

```tsx
import { useSyncExternalStore } from 'react';

function subscribeToStore(store: any, callback: () => void) {
  store.subscribe(callback);
  return () => store.unsubscribe(callback);
}

function useExternalData(store: any) {
  return useSyncExternalStore(
    (callback) => subscribeToStore(store, callback),
    () => store.getState(),
    () => store.getState()
  );
}

function Counter() {
  const count = useExternalData(counterStore);
  return <div>Compteur externe : {count}</div>;
}
```

Ce pattern est particulièrement pertinent pour des bibliothèques de gestion d’état et des intégrations d’API temps réel.

## 5. Optimisations SSR / hydratation

Les applications hybrides profitent d’un rendu côté serveur plus robuste et d’une hydratation progressive.

### Exemple : composant client/serveur

```tsx
'use client';

import { useState } from 'react';

export default function ClientCounter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount((value) => value + 1)}>Count: {count}</button>;
}
```

React 19 améliore le découplage entre le rendu serveur et le rendu client, réduisant les problèmes où l’hydratation se désynchronise.

## 6. Meilleures pratiques de performance

React 19 n’élimine pas le besoin d’écrire du code propre :

- `React.memo` pour les composants purs
- `useMemo` et `useCallback` pour les calculs ou fonctions coûteux
- `lazy` + `Suspense` pour charger les composants à la demande
- `startTransition` pour différer les tâches non urgentes

### Exemple : `React.memo`

```tsx
import { memo } from 'react';

const TagList = memo(function TagList({ tags }: { tags: string[] }) {
  return (
    <ul>
      {tags.map((tag) => <li key={tag}>{tag}</li>)}
    </ul>
  );
});
```

## 7. Accessibilité et conventions React 19

React 19 encourage l’usage de composants accessibles dès la base :

- attributs `aria-*` cohérents
- structure sémantique claire
- gestion clavier correcte

### Exemple d’un bouton accessible

```tsx
function IconButton({ label, onClick }: { label: string; onClick: () => void }) {
  return (
    <button type="button" aria-label={label} onClick={onClick}>
      <span aria-hidden="true">⭐</span>
    </button>
  );
}
```

## Conclusion

React 19 ne change pas les bases, mais affine l’écosystème :

- des hooks plus matures pour l’événementiel et le rendu différé,
- une meilleure intégration SSR,
- plus de stabilité pour les flux externes,
- et des interfaces plus réactives.

Pour bien démarrer, testez `useEvent`, `useDeferredValue` et `useSyncExternalStore` dans des cas concrets, et privilégiez le rendu différé pour les interfaces lourdes.
